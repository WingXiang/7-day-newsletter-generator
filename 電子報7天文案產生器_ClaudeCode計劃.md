## 🎯 產品定位

**目標用戶**：個人創作者、部落客  
**核心功能**：根據用戶輸入的品牌資料，自動產出兩套完整的 7 封 Email 序列文案：
1. **歡迎序列**（Welcome Sequence）— 建立訂閱者信任
2. **銷售序列**（Sales Sequence）— 促進未購買者下單

## ⚙️ 核心功能規格

### 功能一：雙序列分開產生
- 表單送出後，顯示兩個分頁/區塊：「歡迎序列」與「銷售序列」
- 每套序列各產出 7 封 Email
- 每封 Email 包含：
  - **主旨行**（Subject Line）× 1（主要）＋ × 1（備用 A/B test）
  - **預覽文字**（Preview Text）
  - **Email 正文**（完整文案，含段落結構）
  - **CTA 按鈕文字**（Call to Action）

### 功能二：單封重新生成
- 每封 Email 旁有「🔄 重新生成此封」按鈕
- 點擊後，只重新呼叫 AI 生成該封信，其餘 6 封保持不變
- 重新生成時可讓用戶輸入「額外指示」（如：「語氣更幽默」、「強調這個痛點」）

### 功能三：ConvertKit / Kit 格式匯出
- 提供「複製此封」按鈕（複製純文字到剪貼簿）
- 提供「匯出全部序列」按鈕，下載 `.txt` 或 `.md` 檔，格式符合 ConvertKit 序列設定邏輯：
  ```
  === Email 1 ===
  發送時機：訂閱後立即
  主旨：...
  預覽文字：...
  正文：
  ...
  CTA：...
  ```

## 🛠 技術規格

### 技術棧
- **前端**：React（單頁應用）+ Tailwind CSS
- **AI 呼叫**：Anthropic Claude API（`claude-sonnet-4-20250514`）直接從前端呼叫
- **無需後端**：純前端應用，不需要資料庫
- **匯出**：使用 `Blob` API 下載文字檔

### AI Prompt 工程規格

每次呼叫 Claude API 時，system prompt 需包含：
```
你是一位專業的 Email 文案策略師，專門協助個人創作者和部落客撰寫高轉換率的電子報序列。

你的文案原則：
1. 語氣要像朋友寫信，不是廣告文案
2. 每封信只做一件事、傳遞一個核心訊息
3. 主旨行要讓人忍不住點開（好奇、利益、情緒）
4. 結尾 CTA 清晰明確，只有一個行動
5. 使用繁體中文撰寫
```

User prompt 需動態注入用戶填寫的所有欄位資料。

### 狀態管理
```
appState = {
```
